# Compatibility and publishing support

Imposia is a browser-only, React-first HTML/CSS publishing library. `@imposia/core`
can be used directly from browser ESM; `@imposia/client` combines Core and Viewer;
`@imposia/react` is the primary application adapter. There is no Node runtime or
CLI publishing surface.

## Status vocabulary

| Status | Meaning |
| --- | --- |
| Stable | Public, tested behavior in the declared browser contract. Changes are expected to preserve the documented shape and lifecycle. |
| Experimental | Opt-in or draft-heavy behavior. It has explicit limits and can produce a typed fallback warning. |
| Constrained | A stable, intentionally smaller subset. Inputs outside the subset remain atomic or warn; this is not full CSS parity. |
| Unsupported | Outside the product contract. Imposia does not promise an equivalent result or byte artifact. |

## Browser publishing matrix

| Capability | Status | Contract and boundaries |
| --- | --- | --- |
| Browser ESM package surface (`@imposia/core`, `@imposia/client`, `@imposia/react`, `@imposia/viewer`) | Stable | Browser DOM, `iframe`, `Blob`, and `Window.print()` APIs only. Core is usable without React; Client is the framework-neutral convenience entrypoint. |
| Committed HTML/CSR sequence integrity | Stable | For the declared public continuity fixture, every unique source token appears exactly once and in source order after all committed content pages are flattened. The fixture records each page's first token, last token, and count across Chromium, Firefox, and WebKit. Rapid host updates keep the previous canonical generation visible while staging and allow only a complete winning generation to replace it. This is not a claim of lossless fragmentation for arbitrary HTML or every CSS layout; inputs outside the documented subsets remain atomic or emit typed warnings. |
| React adapter lifecycle and imperative handle | Stable | `ImposiaPageViewer` and `ImposiaDocument` retain one Core controller and one canonical iframe. The page Viewer handle exposes `current`, spread presentation controls, `print()`, and `exportEpub()`. `useImposiaDocument()` exposes the host ref, state, current document, and controller for custom presentation. Printing and EPUB export wait for the latest successful Core generation; a failed update preserves the prior commit and destroy aborts pending publishing work. |
| Ordered Publication and Reader navigation | Stable | `mountPublication()` atomically commits metadata, ordered entries, page ranges, content, and one immutable outline. Viewer and React render that outline as a hierarchical table of contents in the existing Page Viewer shell alongside the canonical iframe. Keyboard open, linear movement, Home/End, Escape, selection focus return, current-generation destination validation, and URL-safe stable-ID restore are public behavior. Reader mounting and refresh reject controller/document ownership mismatches before mutating Viewer state; no authored-DOM reparse or second layout authority is used. |
| Publication semantic text search | Stable | `PublicationController.search()` indexes only sanitized visible text in the current committed pages and returns immutable entry metadata, global pages, plain-text excerpts, and controller-and-generation-scoped destinations. Hidden, inert, `aria-hidden`, script, style, and template content is excluded. Reader and React search controls use the existing destination path and canonical iframe; snapshot or controller replacement rejects old result destinations as stale. Destroy clears Reader search state and rejects later Reader actions. No raw DOM or scriptable markup crosses the API. |
| Publication page thumbnails | Stable | Viewer creates one immutable `PublicationThumbnail` for every committed global page from retained `PageMetadata`. A preview preserves sheet aspect ratio and contains at most six abstract text-line marks; it does not clone authored DOM, rasterize pages, create another iframe, or paginate again. Selection targets the exact global page in the canonical Viewer. Generation replacement discards old models and preview nodes, retained stale models are rejected, and destroy removes controls, listeners, and subtrees. |
| Viewer diagnostics Inspector | Stable | `PageViewerOptions.inspector` is opt-in and projects only current-generation `PageWarning` values into a Viewer-owned accessible panel. Findings show code, message, recovery, generation, entry, and page when available. Page findings use the existing global-page path; entry-only findings use the committed entry start, and global findings remain informational. Continuous navigation scrolls the target into view. Core supplies frozen numeric bounds for a trusted current source fragment without exposing a DOM node or retaining a canonical marker, and Viewer draws a temporary non-layout screen outline outside the iframe. Update, presentation synchronization, expiry, and destroy clear highlighting. Inspector, Contents, Search, and Page thumbnails are mutually exclusive. The panel is hidden for print and absent from semantic EPUB export. |
| Page Viewer continuous, single, and spread presentation | Stable | `PageViewerMode` supports all three modes while PDF `ViewerMode` remains continuous/single. `spread.cover` places page 1 alone and pairs 2–3 onward; without it, pairing starts at 1–2. Containers below 720 px preserve the requested spread and global current page but expose and announce a single-page `effectiveMode`; the selected mode control is kept in view in the horizontally scrollable mobile controls. Next, Previous, Arrow, and Page keys advance by one visible spread, while `goToPage()` retains exact global-page targeting. Presentation reuses the canonical iframe and page elements, never creates a second pagination authority, and leaves native print order unchanged. |
| Canonical page DOM, staged updates, isolation, resolver boundary, abort, rollback, and cleanup | Stable | Core owns one persistent sanitized canonical iframe and creates at most one temporary, noncanonical staging iframe per active generation. The commit stays visible while staging resolves and paginates; success atomically replaces the canonical iframe's contents, while failure, abort, or supersession preserves them and removes staging. Both frames use the same restricted sandbox; authored scripts remain disabled by sandbox and CSP. Authored resources never fetch directly; discovered HTML/CSS assets pass through the host `assetResolver`, become Core-owned Blob URLs, and are revoked on replacement, failure, or destroy. |
| Source-aware warning locations | Stable | Every committed `PageWarning` has a frozen `location` with `generation`, `entryId`, and global `page`. Core fills the current generation for every warning, adds a page when its synthetic `sourceIdentity` can be traced to committed content, and adds the owning entry ID for a `PublicationDocument`. Unknown entry or page fields remain explicitly `undefined`. Snapshot replacement exposes only the new current generation through `controller.current.warnings`. Locations never contain authored DOM nodes, raw markup, scripts, asset URLs, resolver output, or temporary source-order markers. `pageWarningTargetBounds()` returns only a frozen rectangle for a current trusted target and returns `undefined` for stale or unresolvable input. |
| Capability-bounded extensions | Stable | Core removes executable markup and normalizes CSS before a PageDocument or Publication transform callback. PageDocument extensions can transform the sanitized copied document strings, filter resolver requests, and return sanitized page decorations. Publication extensions can transform one sanitized copied entry string through `transformEntry` while reading frozen publication and entry metadata; they never receive the composed source or private entry markers. Namespaced diagnostics receive Core-supplied generation, entry, or page provenance. Callback failure rejects with typed `EXTENSION_FAILED`, and abort, supersession, destroy, or failure runs registered cleanup without replacing the committed generation. No extension receives raw unsanitized content, a DOM node, iframe, resolver, Blob URL, mutable warning collection, or unrestricted lifecycle hook. |
| Viewer shell themes | Stable | Consumer CSS modules can override public `--imposia-viewer-*` tokens. `ViewerOptions.theme` and `PageViewerOptions.theme` scope the same tokens to one Viewer; `setTheme()` replaces or clears them without remounting, invalid runtime input is rejected before mutation, and destroy restores pre-viewer host values. Theme tokens never cross Core's iframe isolation into authored document styles. |
| Page presets and custom geometry | Stable | `A4`, `Letter`, portrait/landscape, two absolute dimensions, and one/four-side absolute margins are normalized into `PageGeometry` in CSS pixels at 96 px/in. Host `page` options override authored defaults. |
| Authored `@page` rules and selectors | Stable | Default, `:first`, `:left`, `:right`, `:blank`, and named-page selectors are supported. Supported declarations include `size` and margins; unsupported declarations produce `PAGE_RULE_UNSUPPORTED` rather than a silent approximation. |
| Page margin boxes | Stable | Six boxes are supported: `@top-left`, `@top-center`, `@top-right`, `@bottom-left`, `@bottom-center`, and `@bottom-right`. Supported content is sanitized and projected into the canonical page DOM. |
| Breaks, page parity, `break-inside: avoid`, widows, and orphans | Stable | Nested `break-before`/`break-after`, legacy page-break aliases, page-side constraints, rendered-line breakpoint selection, and deterministic relaxation warnings are covered by the Chromium reference paginator. |
| CJK line breaking, hyphenation, and long tokens | Constrained | Core preserves browser-native, language-tagged horizontal `line-break` and `word-break` behavior and fragments plain-text Korean, Japanese, and Chinese paragraphs at observed rendered-line boundaries without changing source text. Positive integer `widows` and `orphans` constrain page breaks; an engine without computed-property support can use direct or inherited inline integer declarations with `WIDOW_ORPHAN_FALLBACK`, while impossible combinations emit `WIDOW_ORPHAN_RELAXED`. `hyphens: auto` is preserved for a plain-text element with a valid inherited, local, or source-document language; untagged, explicitly empty, or invalid language context falls back to `manual` with `HYPHENATION_FALLBACK`. Dictionary choice and exact hyphen placement remain browser-dependent. An overlong direct text run in an ordinary horizontal wrapping block receives `overflow-wrap: anywhere` with a located `UNBREAKABLE_CONTENT`; authored `white-space: nowrap` remains unchanged and additionally emits `PAGE_OVERFLOW`. Overflowing vertical text remains atomic with `UNSUPPORTED_FRAGMENTATION_CONTEXT` plus `PAGE_OVERFLOW`. |
| Tables | Constrained | Semantic tables fragment between complete row clusters, repeat `thead`, `tfoot`, and authored `colgroup`, and keep `caption` on the first fragment. `createTableColgroupExtension()` is an opt-in preset that freezes measured pixel widths on continued fragments; Core does not synthesize widths by default. Positive bounded `rowspan` cells keep their connected rows together, and `colspan` remains within its row. An oversized row cluster or oversized caption/header/footer remains atomic and emits a located `UNSUPPORTED_LAYOUT` plus `PAGE_OVERFLOW`. Open-ended or cross-group spans, generated table items, internal forced breaks, non-table display, and table-subtree styling that depends on authored ID selectors remain atomic with `UNSUPPORTED_LAYOUT`. |
| Flex and grid | Constrained | Column/no-wrap Flex fragments between in-flow `static` or `relative` children when integer `order` values are nondecreasing in DOM order. Nested Flex cards that meet the same boundary fragment recursively. Fitting `break-inside: avoid` cards move intact; impossible avoidance relaxes with `AVOID_RELAXED`. Block Grid with explicit unnamed columns, row auto-flow, and source-ordered, auto-placed static items fragments only between complete rows; resolved column and source-row track sizes remain stable across continuations, item breaks move the row, and fitting avoid items remain intact. Flex wrapping, reverse directions, non-monotonic order, generated Flex items, and Grid spanning, explicit placement, reordering, dense/column flow, template areas, named/subgrid/masonry tracks, generated items, or nested forced breaks remain atomic with `UNSUPPORTED_LAYOUT`. |
| Multi-column layout | Constrained | A horizontal left-to-right multicol container with zero outer margins, a positive absolute `height` from an active authored rule, `column-fill: auto`, an integer `column-count` or positive absolute `column-width`, and an absolute `column-gap` fragments overflow columns across pages in source order. Static block children may use `break-inside: avoid`; direct `column-span: all` children no taller than the container form full-width boundaries between column lines. Nested or oversized spanners, nested multicol, balanced fill, generated items, list-item roots and list markers, positioned/floating/transformed boxes, nested table/flex/grid, internal forced page breaks or page names, direct text, relative or auto height, container/scope rule dependencies, fragment-sensitive structural selectors, authored ID-selector-dependent styling, and geometry taller than the usable page remain atomic with a located `UNSUPPORTED_LAYOUT`. |
| Local target references and named strings | Constrained | Same-document `#id` links, top-level `::before`/`::after` target page/text markers, and source-order named strings can populate supported margin boxes. Target functions participate in authored `content` importance, specificity, and source order. Marker styling must be co-located with the target `content` declaration; conditional/layered target content and conditional pseudo-content competition are rejected or conservatively suppressed with a typed warning. Missing or duplicate IDs warn deterministically; arbitrary cross-document lookup is not provided. |
| Footnotes and page floats | Experimental | Enable with `experimental.footnotes` or `experimental.pageFloats`. Footnotes are ordered, page-local, bottom-placed, and size/defer limited; page floats are top/bottom only. Disabled, oversized, unsupported, or over-deferred content falls back with `FOOTNOTE_DEFERRED` or `PAGE_FLOAT_FALLBACK`. |
| Browser EPUB export | Stable | `PageDocument.exportEpub()` returns a reflowable EPUB 3.3 `Blob` from the retained semantic source. It emits a deterministic store-mode archive, requires title/language/identifier metadata, admits only retained resolver assets, and enforces entry/byte limits plus abort and destroyed-document checks. Page wrappers, margin furniture, generated counters, Blob URLs, and page-only experimental artifacts are excluded. |
| Native print / Save as PDF | Stable | `PageDocumentController.print()`, the React handle, and the page Viewer clone the accepted canonical pages into a transient shadow-root host in the top document, hoist only required `@page` and `@font-face` rules, and invoke the top `Window.print()`. The browser print dialog can offer Save as PDF. No PDF-byte export API is provided. |
| Fixed-layout EPUB, PDF byte export, Node/CLI rendering, and full CSS parity | Unsupported | Use native browser print for a PDF artifact and the reflowable EPUB Blob for semantic content. A server renderer, command-line renderer, fixed-layout EPUB, or complete implementation of every CSS fragmentation context is outside this browser contract. |

## Browser split

Chromium is the structural pagination reference. Firefox and WebKit are exercised
for the public API, iframe/CSP isolation, resolver boundaries, lifecycle and
cleanup, print invocation, and EPUB archive behavior. Their pagination metrics or
line-breaking details may differ; exact cross-browser page-count or pixel parity is
not promised.

Warnings are part of the public diagnostic surface. A supported subset can still
emit a recovery warning when a hard constraint cannot fit; unsupported authored
syntax is not silently presented as equivalent output. See
[`docs/verification.md`](verification.md) for the test and artifact mapping and
[`docs/architecture/0006-browser-publishing-contract.md`](architecture/0006-browser-publishing-contract.md)
for the auditable decision record.
