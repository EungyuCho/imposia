---
title: 미리보기와 출력이 일치하는 방식
description: Imposia가 어떤 문서를 화면에 보여주고, 인쇄하고, EPUB으로 만드는지 설명합니다.
---

이미지, 표, 페이지가 많은 문서는 페이지를 나누는 데 시간이 걸립니다. Imposia는 다음 문서를 준비하는 동안 마지막으로 완성된 문서를 계속 보여줍니다. 사용자는 덜 만들어진 레이아웃을 보지 않습니다.

## 사용자가 보는 문서

Core는 화면과 인쇄의 기준이 되는 iframe 하나를 계속 유지합니다. 이 문서에서는 이를 canonical iframe이라고 부릅니다. 여기에는 페이지 만들기를 마지막으로 성공한 문서가 들어 있습니다.

React 컴포넌트와 Viewer는 이 iframe을 그대로 보여줍니다. 페이지를 복제하거나 화면 표시용 문서를 따로 만들지 않습니다.

## 내용이 바뀔 때 일어나는 일

1. Core가 새 내용을 위한 임시 iframe을 만듭니다.
2. 허용된 에셋을 불러오고 모든 페이지를 계산합니다.
3. 성공하면 canonical iframe의 내용을 한 번에 바꿉니다.
4. 실패하거나 취소되거나 더 새로운 요청으로 대체되면 임시 iframe을 없애고 이전 문서를 유지합니다.

임시 iframe은 미리보기, 화면 표시, 인쇄에 사용하지 않습니다.

## 출력마다 사용하는 원본

| 출력 | 사용하는 원본 | 결과 |
| --- | --- | --- |
| 미리보기 | canonical iframe | Viewer에 보이는 완성된 페이지 |
| 인쇄 | `Window.print()`를 호출한 canonical iframe | 브라우저의 기본 인쇄 절차. Imposia는 PDF 바이트를 반환하지 않습니다. |
| EPUB | 마지막으로 확정한 문서 원본 | 페이지 DOM의 고정 레이아웃 복사본이 아닌 리플로우형 EPUB 3.3 `Blob` |

`PageDocument.exportEpub()`은 extension을 다시 실행하거나 고정된 페이지 레이아웃을 직렬화하지 않습니다.

## 외부 리소스를 불러오는 경로

외부 리소스는 앱이 제공한 `assetResolver`를 통해서만 들어옵니다. Core는 이 과정에서 만든 Blob URL을 관리하고, 문서를 교체하거나 작업이 실패하거나 인스턴스를 제거할 때 URL을 폐기합니다.

정리된 입력, resolver 결과, extension에는 문서에 적힌 제한이 적용됩니다. 기능을 정확히 적용할 수 없으면 Imposia가 임의로 추측하는 대신 정해진 코드와 메시지가 있는 경고를 남깁니다.

## 지원 범위를 확인하세요

- **안정적으로 지원:** 브라우저 ESM API, iframe 수명 주기, 리소스 격리, 지원되는 페이지 미디어, 브라우저 인쇄, 리플로우형 EPUB.
- **정해진 범위에서 지원:** 문서에 적힌 표, flex, grid, 다단, 참조, named string 기능.
- **실험적으로 지원:** 직접 켜야 하는 페이지 내부 각주와 페이지 float. 이 기능은 경고를 남깁니다.
- **지원하지 않음:** Node·CLI 렌더링, 서버 내보내기, PDF 바이트, 고정 레이아웃 EPUB, 임의의 CSS fragmentation, 브라우저마다 같은 페이지 수를 보장하는 기능.

페이지 구조를 확인할 때는 Chromium을 기준으로 사용하세요. Firefox와 WebKit에서도 공개 API와 문서를 만들고 교체하는 흐름은 동작하지만, 측정값과 줄바꿈은 달라질 수 있습니다.
