Imposia は HTML と CSS をブラウザー内でページ文書に変換します。プレビューと印刷は完成済みページを使い、EPUB はそのページと同時に確定したセマンティックソースから作成します。

## 目的から選ぶ

- **まずプレビューを動かす：** [最初のページを作る](/ja/docs/getting-started)に沿って React パッケージをインストールし、HTML を A4 ページとして表示します。
- **出力の仕組みを理解する：** 印刷、EPUB ダウンロード、外部アセットを組み込む前に、[プレビューと出力が一致する仕組み](/ja/docs/publishing-contract)を確認してください。

## HTML がページになるまで

1. アプリから HTML、CSS、ページ設定を Imposia に渡します。
2. Core が一時 iframe で次のバージョンをページ分割します。
3. ページ分割が完了すると、Imposia が表示中の文書を一度に置き換えます。

ページ分割に失敗した場合や、より新しい更新が先に完了した場合も、読者には直前の完成済み文書が表示されます。未完成のレイアウトが見えることはありません。

## 1つの確定済みソースを3つの用途に

- **プレビュー：** React Viewer で完成済みページを表示します。
- **印刷：** 同じページを対象にブラウザーの印刷ダイアログを開きます。
- **EPUB：** 文書の最新のコミット済みセマンティックソースから、リフロー型 EPUB 3.3 `Blob` を作成します。

## 対応範囲を確認する

Imposia はブラウザー内で動作し、ESM として提供されます。Node レンダラー、CLI レンダラー、サーバーパブリッシング API、PDF ジェネレーターは含まれません。この構成がアプリに合う場合は、[最初のページを作る](/ja/docs/getting-started)へ進んでください。
