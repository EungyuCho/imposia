Imposia는 HTML과 CSS를 브라우저 안에서 페이지로 만듭니다. 미리보기와 인쇄는 완성된 페이지를 사용하고, EPUB은 그 페이지와 함께 확정된 의미 구조에서 만듭니다.

## 먼저 읽을 문서를 고르세요

- **페이지를 바로 만들어 보고 싶다면:** [첫 페이지 만들기](/ko/docs/getting-started)에서 React 패키지를 설치하고 HTML을 A4 페이지로 표시해 보세요.
- **출력 방식부터 이해하고 싶다면:** 인쇄, EPUB 다운로드, 외부 에셋을 연결하기 전에 [미리보기와 출력이 일치하는 방식](/ko/docs/publishing-contract)을 읽어보세요.

## HTML을 넘기면 이렇게 처리합니다

1. 앱이 HTML, CSS, 페이지 설정을 Imposia에 전달합니다.
2. Core가 임시 iframe에서 다음 문서의 페이지를 계산합니다.
3. 모든 페이지를 만든 뒤 화면에 보이는 문서를 한 번에 바꿉니다.

페이지를 만드는 도중 오류가 나거나 더 새로운 요청이 들어오면 이전 문서를 그대로 보여줍니다. 사용자는 작업 중인 레이아웃을 보지 않습니다.

## 같은 원본을 세 가지 방식으로 사용합니다

- **미리보기:** React Viewer에서 완성된 페이지를 보여줍니다.
- **인쇄:** 화면에 보이는 페이지로 브라우저 인쇄 창을 엽니다.
- **EPUB:** 마지막으로 확정된 의미 구조에서 리플로우형 EPUB 3.3 `Blob`을 만듭니다.

## 시작 전에 범위를 확인하세요

Imposia는 브라우저에서 ESM으로 실행됩니다. Node 렌더러, CLI 렌더러, 서버 퍼블리싱 API, PDF 생성기는 포함하지 않습니다. 이 조건이 앱의 요구사항과 맞다면 [첫 페이지 만들기](/ko/docs/getting-started)로 이어가세요.
